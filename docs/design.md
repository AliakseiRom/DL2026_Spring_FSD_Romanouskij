# Часть 1. Проектирование и анализ

## 1.1. Пользовательские сценарии

**Сценарий 1:**  
*Как пользователь, я хочу авторизоваться или зарегистрироваться в приложении, чтобы получить доступ к персонализированной информации о погоде и мемах.*

**Сценарий 2:**  
*Как пользователь, я хочу видеть текущую погоду в выбранном городе и соответствующий мем, чтобы получить полезную информацию и развлечение.*

**Сценарий 3 (для администратора):**  
*Как администратор, я хочу иметь доступ к панели управления, чтобы добавлять, редактировать или удалять мемы и управлять контентом приложения.*

---

## 1.2. Функциональные требования

### Обязательные:
- Регистрация и авторизация пользователей с JWT.
- Отображение погоды для выбранного города.
- Отображение мемов, соответствующих погоде.
- Редирект пользователя по роли (обычный пользователь → `/weather`, админ → `/admin`).
- Защита маршрутов через ProtectedRoute и роли.

### Опциональные:
- Возможность смены города без перезагрузки страницы.
- Фильтрация мемов по категории или настроению.
- Поддержка нескольких языков интерфейса.

---

## 1.3. Проектирование API

| Метод | Путь | Параметры | Описание | Авторизация |
|-------|------|-----------|----------|-------------|
| POST  | /auth/register | `{ username, email, password }` | Регистрация нового пользователя | нет |
| POST  | /auth/login | `{ username, password }` | Авторизация, возвращает JWT | нет |
| GET   | /weather?city={city} | city (query) | Получение данных о погоде и мемов | да (JWT) |
| GET   | /admin/memes | - | Получение списка мемов для админ-панели | да (JWT + роль ADMIN) |
| POST  | /admin/memes | `{ title, url, weatherType }` | Добавление нового мема | да (JWT + роль ADMIN) |
| PUT   | /admin/memes/{id} | `{ title, url, weatherType }` | Редактирование мема | да (JWT + роль ADMIN) |
| DELETE| /admin/memes/{id} | - | Удаление мема | да (JWT + роль ADMIN) |

---

## 1.4. Модель данных

### Сущность `User`  
| Поле        | Тип        | Описание |
|------------|------------|----------|
| id         | Long       | Уникальный идентификатор |
| username   | String     | Имя пользователя |
| email      | String     | Email пользователя |
| password   | String     | Хэшированный пароль |
| role       | Enum       | Роль пользователя (USER, ADMIN) |

### Сущность `Meme`  
| Поле        | Тип        | Описание |
|------------|------------|----------|
| id         | Long       | Уникальный идентификатор |
| title      | String     | Заголовок мема |
| url        | String     | Ссылка на изображение мема |
| weatherType| String     | Тип погоды, к которой относится мем (например: солнечно, дождь, снег) |

Пример JSON для Meme:  
```json
{
  "id": 1,
  "title": "Funny Rain Meme",
  "url": "https://example.com/meme1.png",
  "weatherType": "rain"
}
