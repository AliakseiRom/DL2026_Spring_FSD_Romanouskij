# Часть 3. Работа с AI

## 3.1. Применённые инструменты и промпты

В ходе выполнения проекта использовались следующие AI-инструменты:

- **ChatGPT (OpenAI GPT-5)**  
  Применялся для генерации и корректировки кода, объяснения ошибок, написания документации и планирования структуры проекта.  

  **Примеры промптов:**
  1. *"Сделай ProtectedRoute для React с использованием AuthContext и TailwindCSS."*  
     → Использовалось для реализации безопасной маршрутизации на фронтенде.
  2. *"Объясни ошибку `Invalid hook call` в React и как её исправить."*  
     → Помогло понять причину ошибки и исправить неправильное использование хуков.
  3. *"Составь документацию проекта в формате Markdown по плану: проектирование и анализ."*  
     → Использовалось для быстрого создания структурированной документации.

---

## 3.2. Анализ эффективности

**Где AI был полезен:**
- Генерация шаблонного кода (ProtectedRoute, AuthContext, формы авторизации/регистрации).  
- Быстрое исправление ошибок и предоставление пошаговых объяснений.  
- Создание документации в структурированном виде и генерация Markdown-файлов.  

**Где ответы AI требовали корректировки:**
- Иногда предложения по структуре компонентов React были не полностью совместимы с текущей архитектурой проекта (например, неправильное использование хуков вне компонентов).  
- Примеры работы с JWT иногда включали лишние шаги или устаревшие методы. Требовалась ручная проверка и адаптация к текущему стеку (React 18, TailwindCSS, React Router 6).  
- SQL-запросы в `data.sql` AI мог предложить так, что они удаляли все данные при каждом запуске. Пришлось вручную адаптировать скрипт для безопасной инициализации базы.

**Примеры ситуаций:**
- **Ускорение разработки:** генерация AuthContext и ProtectedRoute, где AI предоставил почти готовый код, оставалось только вставить в проект.  
- **Ошибка AI:** пример использования хуков вне компонента, что вызвало ошибку `Invalid hook call`. Обнаружено в браузере, исправлено вручную.  
- **Отказ от предложения AI:** несколько раз отказывался от предложений по полям модели данных и SQL, так как они не соответствовали требованиям безопасности или логике приложения.

---

## 3.3. Выводы и рекомендации

1. **AI ускоряет рутинную разработку:** генерация шаблонного кода, документации и повторяющихся компонентов позволяет экономить время.  
2. **Проверка и адаптация обязательны:** предложения AI всегда требуют ручной проверки, особенно при работе с безопасностью, хуками и актуальной версией стека.  
3. **Планомерное использование:** для архитектурного планирования, пояснений ошибок и создания документации AI помогает систематизировать работу.  

**Рекомендации для коллег:**
- Использовать AI как ассистента, а не как окончательный источник кода.  
- Проверять предложенный код и адаптировать под свой проект.  
- Для документации и прототипов AI может быть основным инструментом ускорения работы.
